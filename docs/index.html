Features

Everything you need to keep your code mirrored and safe.

↔

Multi-Provider Sync

Push to GitHub, GitLab, Gitea, and generic Git servers simultaneously with a single command.

🔒

Secure Credentials

OS keychain, SSH keys, git helpers, environment variables, or encrypted file. Tokens never stored in plaintext config.

⚡

Auto-Sync Hooks

Install a post-push hook and every git push automatically mirrors to all providers. No extra commands.

📦

Multi-Repo Management

Register dozens of repositories and push them all at once with copygit push --all.

⚠

Conflict Detection

Detects diverged branches with ahead/behind counts. Warn, merge, or force-push strategies.

🤖

Background Daemon

Continuous polling-based sync runs in the background. Start, stop, and check status anytime.

Quick Start

Get mirroring in under a minute.

1
Add your providers — Define where you want to mirror. GitHub, GitLab, Gitea, or any Git server.
2
Authenticate — Store credentials securely. Tokens are hidden and stored in your OS keychain.
3
Register a repo — Point CopyGit at any local Git repository and select which providers to sync.
4
Push everywhere — One command mirrors your code to all configured providers.
5
Automate it — Install a hook so every git push mirrors automatically.
# Add providers
copygit config add-provider github github https://github.com
copygit config add-provider gitlab gitlab https://gitlab.com

# Authenticate (token stored in OS keychain)
copygit login --provider github
copygit login --provider gitlab

# Register your repo
cd ~/projects/my-app
copygit init .

# Push to all providers
copygit push

# Auto-sync on every git push
copygit hooks install

Commands

Full CLI reference.

Core
copygit init <path>Register a repo for syncing. Auto-detects remotes, prompts for provider selection, creates .copygit.toml.
copygit push [path]Push branches and tags to all configured providers. Use --all for all repos, --dry-run to preview.
copygit sync [path]Full bidirectional sync: fetch all remotes, detect conflicts, then push. Use --force to push despite divergence.
copygit status [path]Show sync state per provider (in-sync, ahead, behind). Use --all for all repos.
copygit listList all registered repositories with their last sync time.
copygit remove <path>Unregister a repo. Use --clean to also delete .copygit.toml.
Configuration
copygit config add-provider <name> <type> <url>Add a provider. Types: github, gitlab, gitea, generic.
copygit config list-providersShow all configured providers and their auth methods.
copygit config remove-provider <name>Remove a provider from the global configuration.
copygit login --provider <name>Authenticate with a provider. Interactive token prompt (hidden input). Validates before storing.
Automation
copygit hooks install [path]Install a post-push hook that mirrors on every git push.
copygit hooks uninstall [path]Remove CopyGit's hook from the repo. Safe: preserves other hooks.
copygit hooks status [path]Show whether the auto-sync hook is installed.
copygit daemon startStart background sync daemon. Polls all repos at a configurable interval (default: 30s).
copygit daemon stopStop the running daemon via PID file signal.
copygit daemon statusCheck if the daemon is running.

Configuration

Three files, clean separation of concerns.

~/.copygit/config Global 
# Provider definitions (shared across all repos)

[providers.github]
name       = "github"
type       = "github"
base_url   = "https://github.com"
auth_method = "https"
preferred  = true

[providers.gitlab]
name       = "gitlab"
type       = "gitlab"
base_url   = "https://gitlab.com"
auth_method = "token"

[daemon]
poll_interval = "30s"
<repo>/.copygit.toml Per-repo 
# Which providers this repo syncs to

version = "1"

[[sync_targets]]
provider   = "github"
remote_url = "git@github.com:user/repo.git"
enabled    = true

[[sync_targets]]
provider   = "gitlab"
remote_url = "git@gitlab.com:user/repo.git"
enabled    = true
~/.copygit/repos.toml Registry 
# Tracks all registered repos

version = "1"

[[repos]]
path          = "/home/user/projects/api"
alias         = ""
registered_at = "2026-03-01T10:00:00Z"
last_sync     = "2026-03-06T17:00:00Z"

[[repos]]
path          = "/home/user/projects/web"
alias         = ""
registered_at = "2026-03-01T10:05:00Z"
last_sync     = "2026-03-06T16:55:00Z"

Credential Resolution Chain

CopyGit tries each source in order until it finds a valid credential.

1
OS Keychain
macOS Keychain, Linux Secret Service, Windows Credential Manager
2
SSH Keys
~/.ssh/id_ed25519, id_rsa, id_ecdsa
3
Git Credential Helper
git credential-store, git credential-cache, GCM
4
Environment Variable
COPYGIT_TOKEN_<PROVIDER_NAME>
5
Encrypted File
~/.copygit/credentials (enforced 0600 permissions)

Architecture

Clean layered design with interface-driven components.

cmd/copygit/ CLI entry point + Cobra commands command/ init, push, sync, status, list, remove, config, hooks, daemon internal/ model/ Domain types (ProviderConfig, Credential, SyncTarget, ConflictInfo) config/ TOML config management (global + per-repo + registry) credential/ Multi-source credential chain (keyring, SSH, env, file, githelper) git/ Git CLI wrapper (executor, remote, branch, tag, status) provider/ Provider adapters (GitHub, GitLab, Gitea, generic) with registry sync/ Orchestrator for push/fetch operations with conflict detection output/ Formatters (text, JSON) for human and machine output lock/ Per-repo file locking (flock on Unix, LockFileEx on Windows) hook/ Git hook management (marker-based insertion, safe coexistence) daemon/ Background polling daemon with PID file management

Native Git

Wraps the git CLI via os/exec. No custom protocols, no lock-in. Your repos stay standard Git repos.

Interface-Driven

GitExecutor, Provider, CredentialResolver, and Formatter are all interfaces with test fakes.

Per-Repo Locking

File-based locks prevent concurrent syncs on the same repository. Platform-specific: flock on Unix, LockFileEx on Windows.
